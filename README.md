# TRON UCP x402 Demo

Local demo for two payment flows on TRON Nile:

- Bank of AI x402 for `GET /api/premium-data`
- UCP checkout sessions for `/ucp/v1/checkout-sessions`

Repository: `https://github.com/aziz1975/tron-ucp-x402-demo.git`

## What Runs Here

This repo starts three local processes:

1. `x402_service.py`
   Bank of AI FastAPI middleware service that generates the x402 `402 Payment Required` response for the premium route.
2. `server.js`
   Express app that serves the UCP APIs, proxies `/api/premium-data` to the x402 middleware service, stores orders in `orders.json`, and handles Telegram or mock approval for the UCP checkout flow.
3. `frontend`
   Vite dashboard for viewing orders and triggering the demo agent.

Important behavior:

- `/api/premium-data` uses Bank of AI x402 and does not use Telegram 2FA.
- `/ucp/v1/checkout-sessions` uses the repo's UCP flow and can use Telegram 2FA or local mock approval.

## Requirements

- Node.js `22+`
- npm `10+`
- Python `3.10+`
- A TRON Nile wallet for `MERCHANT_ADDRESS`
- A TRON Nile test wallet private key for `TRON_PRIVATE_KEY`
- Nile test TRX for the agent wallet
- Nile test TRC20 USDT for the agent wallet

Optional:

- Telegram bot token and chat ID if you want real approval prompts for the UCP checkout flow

## Quick Start

### 1. Clone

```bash
git clone https://github.com/aziz1975/tron-ucp-x402-demo.git
cd tron-ucp-x402-demo
```

### 2. Install Dependencies

Root dependencies:

```bash
npm install
```

Frontend dependencies:

```bash
cd frontend
npm install
cd ..
```

Python dependencies for the Bank of AI middleware service:

```bash
python3 -m venv .venv-x402
./.venv-x402/bin/pip install -r requirements-x402-service.txt
```

### 3. Configure Environment

Create the local env file:

```bash
cp .env.example .env
```

Set the required values in `.env`:

```env
PORT=8000
MERCHANT_ADDRESS=TYourMerchantWalletAddress
TRON_PRIVATE_KEY=your_nile_test_agent_private_key
TRC20_USDT_CONTRACT=TXYZopYRdj2D9XRtbG411XZZ3kM5VkAeBf

FACILITATOR_URL=https://facilitator.bankofai.io
X402_NETWORK=tron:nile
X402_SERVICE_URL=http://localhost:8001
X402_SERVICE_PORT=8001
X402_SERVICE_RESOURCE_PATH=/premium-data
X402_PRICE_DECIMAL=15.00
X402_PRICE_CURRENCY=USDT
X402_SERVICE_SCHEMES=exact_permit

TELEGRAM_BOT_TOKEN=your_bot_token
TELEGRAM_CHAT_ID=your_chat_id
```

Minimum required for the x402 premium flow:

- `MERCHANT_ADDRESS`
- `TRON_PRIVATE_KEY`
- `FACILITATOR_URL`
- `X402_SERVICE_URL`

Telegram values are optional unless you want to test the UCP checkout approval flow.

### 4. Check Wallet Funding

Before running the demo agent, confirm the buyer wallet has Nile test funds:

```bash
node check-balance.js
```

Expected:

- prints the agent wallet address
- prints TRX balance
- prints TRC20 USDT balance

## Test Flow

Use this order so you can isolate problems quickly.

### A. Smoke Test the x402 Middleware Service

Unpaid request directly to the Python service:

```bash
curl -i http://localhost:8001/premium-data
```

Expected:

- HTTP `402 Payment Required`
- x402 challenge in the response

This `402` is generated by Bank of AI middleware, not by `server.js`.

### B. Smoke Test the Proxied Premium Route

Same request through Express:

```bash
curl -i http://localhost:8000/api/premium-data
```

Expected:

- HTTP `402 Payment Required`
- same x402-style challenge forwarded through the proxy

### C. Run the Demo Agent

```bash
node test-agent.js
```

Expected flow:

1. first request receives `402`
2. `@bankofai/x402` signs the payment payload
3. client retries with `PAYMENT-SIGNATURE`
4. facilitator verifies and settles
5. response returns `200`
6. the agent prints the premium payload

### D. Confirm Settlement Was Recorded

```bash
curl http://localhost:8000/api/orders
```

Expected:

- a new order with `payment_protocol: bankofai_x402`
- `status: PAID`
- a `txHash` if settlement succeeded

### E. Check the Dashboard

Open `http://localhost:5173` and verify:

- the dashboard loads
- the Payments view shows the new order
- the `Run Live Demo Agent` button triggers the same flow visually

## Test the UCP Checkout Flow

This is separate from x402 and still uses the repo's checkout session APIs.

### 1. Create a Checkout Session

```bash
curl -X POST http://localhost:8000/ucp/v1/checkout-sessions \
  -H 'Content-Type: application/json' \
  -d '{"line_items":[{"item":{"id":"premium_data_access"},"quantity":1}]}'
```

Expected:

- HTTP `201`
- order status starts as `AWAITING_2FA`

### 2. Approve the Session

If Telegram is configured, approve from Telegram.

If Telegram is not configured, the server falls back to local mock approval:

```bash
curl -X POST http://localhost:8000/api/demo/approve-2fa/<checkout_session_id>
```

Expected:

- order moves to `PENDING`

### 3. Poll the Session

```bash
curl http://localhost:8000/ucp/v1/checkout-sessions/<checkout_session_id>
```

Expected:

- transfer state becomes `ready_for_transfer`

### 4. Complete with a Real TRON Transaction Hash

```bash
curl -X POST http://localhost:8000/ucp/v1/checkout-sessions/<checkout_session_id>/complete \
  -H 'Content-Type: application/json' \
  -d '{"transactionHash":"<real_tron_tx_hash>"}'
```

Expected:

- server verifies recipient, amount, and contract
- order becomes `PAID`

## Troubleshooting

### `curl http://localhost:8001/premium-data` does not return `402`

Check:

- the Python service is running
- `.env` is present
- `MERCHANT_ADDRESS` is valid
- Python dependencies installed successfully in `.venv-x402`

### `node test-agent.js` fails with insufficient balance

Fund the buyer wallet with:

- Nile TRX
- Nile TRC20 USDT

Use:

```bash
node check-balance.js
```

### `/api/premium-data` returns `502`

That means Express cannot reach `x402_service.py`.

Check:

- `X402_SERVICE_URL`
- `X402_SERVICE_RESOURCE_PATH`
- the Python service is actually listening on port `8001`

### Telegram approval is not appearing

Telegram only applies to the UCP checkout flow, not the x402 premium route.

Check:

- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_CHAT_ID`
- your bot has received `/start`

```bash
curl -X POST http://localhost:8000/api/demo/approve-2fa/<checkout_session_id>
```
